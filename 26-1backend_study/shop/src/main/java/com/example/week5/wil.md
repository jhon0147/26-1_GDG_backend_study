# 1. 객체 지향 프로그래밍(OOP)
프로그램을 묶음 단위로 잘게 쪼개서 조립하듯 전체 프로그램을 완성.

## 객체 지향의 네가지 특징

- 추상화: 객체의 공통적인 속성과 기능을 추출하여 정의

- 캡슐화: 서로 연관있는 속성과 기능들을 하나의 캡슐로 만들어 데이터를 외부로부터 보호

- 상속: 기존 클래스의 속성과 기능을 새로운 클래스가 물려받아 재사용

- 다형성: 어떤 객체의 속성이나 기능이 상황에 따라 여러가지 형태를 가짐


# 2. SOLID의 5가지 원칙

- SRP: 단일 책임 원칙

클래스를 변경하는 이유는 단 하나여야 함. 프로그램의 유지보수성 향상.

- OCP: 개방-폐쇄 원칙

기존 코드를 변경하지 않고 기느을 확장할 수 있도록 설계하는 것.

- LSP: 리스코프 치환 원칙

자식 클래스는 언제나 부모 클래스를 대체할 수 있어야 함. 문법적인 상속, 타입 호환성 그리고 행동적인 호환성까지 보장되어야 한다는 뜻.

- ISP: 인터페이스 분리 원칙

클라이언트가 사용하지 않는 인터페이스에 의존하지 않도록 함. 여러개의 작은 인터페이스를 분리하는 것이 좋음.

- DIP: 의존관계 역전 원칙

구현클래스에 의존하지 않고, 인터페이스에 의존해야 함. 

# 3. IoC
객체 새성 및 관리에 대한 제어권을 개발자가 아닌 프레임워크가 대신 가져가는 것.
->스프링 컨테이너가 수행.

## 스프링 컨테이너
객체를 하나만 생성해서 필요할 때마다 재사용(싱클톤 패턴)->메모리를 효율적으로 사용.

## DI 의존성 주입
IoC를 구현하는 방법.
스프링 컨테이너에 필요한 객체를 미리 생성해두고, 다른 객체에서 이 객체가 필요할 때 주입하는 것.
